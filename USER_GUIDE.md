# 用户使用指南

欢迎使用"本地指南"。这是一个帮助你觉察自己、整理思绪、做出决定的工具。

## 如何开始

打开应用后，你会看到首页，上面有五个地点卡片：
- 共情小屋
- 状态观测台
- 微习惯工坊
- 优先级决策岛
- 回顾花园

**建议从共情小屋或状态观测台开始**，因为它们更适合"先停下来看看自己"。

---

## 五大模块使用说明

### 1. 共情小屋

**适合场景：**
- 情绪有点乱、有点烦躁
- 需要整理自己的想法
- 想先暂停、理清楚再决定下一步

**如何使用：**
1. 描述让你感到困扰的情境（客观描述，不是抱怨）
2. 说出你的感受（不用分析原因）
3. 表达这个情境下你可能的需求
4. 获得一些温和的自我对话建议

**做完后：**
- 记录会保存在本地
- 会出现"下一站"建议，可以去优先级决策岛

**推荐下一站：** 优先级决策岛

---

### 2. 状态观测台

**适合场景：**
- 想记录今天的整体状态
- 需要了解自己的能量和压力水平
- 简单打卡，不需要深度整理

**如何使用：**
1. 用1-10分记录你今天的**能量水平**（1=很低，10=充满活力）
2. 用1-10分记录你今天的**压力水平**（1=很轻松，10=压力很大）
3. 可以添加简短备注

**做完后：**
- 记录会保存在本地
- 可以直接去其他地点

**推荐下一站：** 任意地点，取决于你想做什么

---

### 3. 微习惯工坊

**适合场景：**
- 想培养一个习惯但总失败
- 需要把大目标缩小到今天能做的一小步
- 需要一点视觉反馈和激励感

**如何使用：**
1. 输入你想养成的习惯（建议很小、很具体）
2. 设置频率（每天/每周几次）
3. 完成时点击"完成今天"，会获得温和的视觉反馈

**做完后：**
- 完成记录会保存在本地
- 会显示温暖的"点灯"反馈
- 可以去回顾花园看看自己的坚持

**推荐下一站：** 回顾花园

---

### 4. 优先级决策岛

**适合场景：**
- 有多件事要做，不知道从哪开始
- 纠结要不要做某件事
- 需要一个结构化的方法来决定优先级

**如何使用：**
这是一个"五道门"的决策流程，帮助你一步步找到最合适的行动：

1. **放下门** - 问自己：这件事真的需要做吗？
   - 如果可以删除，那就放下

2. **简化门** - 问自己：能不能把它缩小成更小的版本？
   - 如果可以简化，试试最小可行版本

3. **求助门** - 问自己：这件事必须我来做吗？
   - 如果可以委托或求助，就去做

4. **稍后门** - 问自己：这件事真的现在要做吗？
   - 如果可以推迟，就先做更重要的事

5. **专注门** - 确认这是我现在要做的最重要的事

**做完后：**
- 决策记录会保存在本地
- 会显示你经过的路径
- 会引导你去微习惯工坊，把决定变成具体行动

**推荐下一站：** 微习惯工坊

---

### 5. 回顾花园

**适合场景：**
- 想看看自己记录了什么
- 需要回顾最近的行动和决定
- 想整理和导出数据

**功能：**

**查看记录：**
- 记录按时间分组：今天、昨天、近三天、更早
- 每条记录都有类型标签（共情/状态/习惯/决策）
- 不同类型有不同的颜色标记

**理解类型标签：**
- 共情：来自共情小屋的记录
- 状态：来自状态观测台的记录
- 习惯：来自微习惯工坊的完成记录
- 决策：来自优先级决策岛的决定

**数据管理：**
- **导出JSON**：保存所有数据为备份文件
- **导入JSON**：从备份文件恢复数据
- **清空数据**：删除所有本地记录（会二次确认）

**推荐下一站：** 可以重新从共情小屋开始新一轮探索

---

## 推荐使用顺序

### 情绪困扰时
```
共情小屋 → 优先级决策岛 → 微习惯工坊
```

### 日常打卡时
```
状态观测台 → 微习惯工坊
```

### 定期回顾时
```
回顾花园
```

---

## 关于数据

**你的数据只保存在本地浏览器中**
- 不会上传到任何服务器
- 换浏览器或换设备不会看到之前的数据
- 清空浏览器缓存会丢失数据

**重要：请定期使用"导出JSON"功能备份数据**
