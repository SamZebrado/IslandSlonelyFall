# P2 Final Scope Document

## P2 目标
把当前产品从"功能和体验成形的 P1 原型"推进到"本地状态可靠、视觉层次清楚、更像'本地指南'的稳定版"。

## P2 已实现内容

### P2A - 数据可靠性与恢复机制

**统一存储层（storage.js）**
- schemaVersion = 2，支持版本迁移
- normalizeState() 统一数据格式
- normalizeRecord() 按类型规范化记录
- 缺字段补默认值
- 坏记录隔离（返回null而非崩溃）

**备份恢复**
- BACKUP_KEY 保存最近一次备份
- 主数据损坏时自动从 backup 恢复
- createDefaultState() 安全初始化

**导入失败保护**
- Temp backup 在导入前保存当前状态
- 导入失败时自动回滚
- 明确错误消息返回

**多标签页支持**
- initMultiTabListener(callback) 注册 storage 事件监听
- 外部标签页更新时触发回调
- 跨标签页状态同步感知

**验证报告**
- P2A_TAIL_REPORT.md 详细记录测试场景

### P2B - 本地指南展示增强

**地点感 - 首页地点卡片**
- 五个地点：共情小屋、状态观测台、微习惯工坊、优先级决策岛、回顾花园
- 每个地点有专属 badge（小屋/观测台/工坊/岛屿/花园）
- 每个地点有描述文案和"适合..."标签
- 每个地点有专属色调（暖米色/浅蓝/浅草绿/浅橙/浅粉）
- 副标题："带你慢慢走，每一步都是探索"

**路径感 - 下一站导航**
- 共情记录完成后：引导去优先级决策岛
- 优先级决策完成后：引导去微习惯工坊
- 设计：虚线边框、柔和渐变背景、轻量"前往"按钮

**回顾花园可读性增强**
- 按时间分组（今天/昨天/近三天/更早）
- 每条记录有类型标签和颜色区分
- 顶部显示总记录数和今日数据 summary
- 空花园状态显示引导文案
- 记录卡片 hover 微动效
- 每种记录类型左侧有颜色边线

**柔和反馈**
- 不做分数、等级、连击
- 不做强打卡感
- 强调"留下痕迹"而非"完成任务"

## P2 明确不再扩展

- 不新增决策分支
- 不新增玩法
- 不做复杂游戏化

### 技术边界
- 不做云同步
- 不做账号体系
- 不做大范围信息架构重构
- 不做数据库迁移

### 展示边界
- 不做复杂图表/dashboard
- 不做炫技型动画
- 不做强奖励系统

## 已知限制

1. **备份只存一版**：只保存最近一次 backup，无法恢复更早版本
2. **多标签页不合并**：外部更新直接替换本地状态，无法合并冲突
3. **无自动迁移**：schemaVersion 变化时全量替换，无法渐进迁移
4. **localStorage 依赖**：数据完全依赖浏览器 localStorage，清缓存会丢失
5. **无网络备份**：无法跨设备同步
